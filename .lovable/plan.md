

# Intelligent Clone and Adapt Wizard for Pricing Proposals

## Overview

A multi-step wizard that allows users to select an existing (base) pricing proposal, describe the differences for a new deal, and have AI intelligently adapt the base proposal into a first draft for the new transaction. The wizard combines structured questions, free-text/voice input, and AI-powered adaptation with a review-and-approve workflow before generating the final draft.

---

## User Flow (5 Steps)

### Step 1: Select Base Proposal and New Deal Basics
- User clicks "New from Precedent" button on the Pricing list page (alongside the existing "New Proposal" button)
- A full-page wizard opens (new route: `/pricing/adapt`)
- User selects the base proposal from a dropdown of existing proposals
- User selects or creates the client for the new deal
- User enters the new proposal name
- A read-only summary card shows the base proposal: client, phases, item count, total fee range, currency, technology/sector if discernible from the name

### Step 2: Structured Questions (AI-Powered Targeted Questions)
The wizard presents a series of targeted questions about how the new deal differs from the base. These are generated dynamically by AI after analysing the base proposal's work items, scope assumptions, and phases. Examples of the kinds of questions the AI would generate:

- **Jurisdiction**: "The base proposal relates to [Egypt]. What country/jurisdiction is the new deal in?"
- **Technology/Sector**: "The base proposal involves [solar]. What technology does the new deal involve?" (with common options: wind, solar, battery storage, hydrogen, etc.)
- **Financing**: "The base proposal includes financing-related work items. Does the new deal involve financing?" (Yes/No/Unsure)
- **Regulatory**: "The base proposal includes regulatory filings in [Egypt]. Will the new deal require regulatory work?" (Yes/No/Different jurisdiction)
- **Number of parties**: "The base proposal assumes [X] counterparties. How many are expected in the new deal?"
- **Local counsel**: "The base proposal includes local counsel in [countries]. Which jurisdictions need local counsel for the new deal?"
- **Scope additions/removals**: "Are there any major workstreams in the base proposal that are NOT needed for the new deal?" (checklist of base phases)
- **New workstreams**: "Are there any new workstreams needed that aren't in the base proposal?"

Each question has structured input (dropdowns, checkboxes, text fields) and a "Skip" option. The questions are generated by an edge function that analyses the base proposal content.

### Step 3: Free-Form Description (Text and Voice)
- A large text area where the user can describe any other differences in their own words
- A microphone button that uses browser speech recognition (`webkitSpeechRecognition` / `SpeechRecognition` API) for voice-to-text input -- no external API needed, works natively in Chrome/Edge
- The transcribed speech appends to the text area so the user can review and edit
- Prompt text: "Describe any other differences between the base deal and the new deal. You can type or use the microphone to speak naturally."

### Step 4: AI Adaptation -- Review and Approve
- User clicks "Generate Draft" -- a loading state shows while the AI processes
- An edge function (`adapt-pricing-proposal`) receives: base proposal data (all items, phases, rate card, scope assumptions), structured answers from Step 2, and free-form text from Step 3
- The AI returns a list of proposed changes, each categorised as:
  - **Modified**: item text/detail changed (shows diff: old text vs new text)
  - **Removed**: item recommended for removal (with rationale)
  - **Added**: new item suggested (with rationale)
  - **Unchanged**: kept as-is
- The review page displays these in a clear list grouped by phase, with each change showing:
  - The action (Modified/Removed/Added/Unchanged) as a colour-coded badge
  - The original text (for Modified/Removed) and new text
  - AI rationale for the change
  - Accept/Reject toggle per change (all default to Accept)
  - A comment field per change for user notes
- Scope assumptions changes are shown in a separate section with the same accept/reject pattern
- Phase-level changes (renamed, added, removed) shown at the top
- A "Comment on all" text box at the bottom for general feedback

### Step 5: Second Pass and Final Draft
- After the user has accepted/rejected changes and added comments, they click "Refine and Create"
- The edge function runs a second pass incorporating the user's accept/reject decisions and comments
- A final confirmation screen shows the complete new proposal summary (phases, item count, estimated total)
- User clicks "Create Proposal" to generate the actual `pricing_proposals` record with all items, phases, rate card (copied from base), and adapted scope assumptions
- The user is navigated to the new proposal's detail page

## Technical Plan

### Database Changes
No new tables required. The wizard creates a standard `pricing_proposals` record and `pricing_proposal_items` via the existing mutation infrastructure. The wizard state is held entirely in React component state until final creation.

### New Files

1. **`src/pages/AdaptPricingWizard.tsx`** -- Main wizard page with step navigation, state management across all 5 steps
2. **`src/components/pricing/adapt/StepSelectBase.tsx`** -- Step 1: base proposal selector and new deal basics
3. **`src/components/pricing/adapt/StepStructuredQuestions.tsx`** -- Step 2: AI-generated targeted questions
4. **`src/components/pricing/adapt/StepFreeDescription.tsx`** -- Step 3: text area + voice input
5. **`src/components/pricing/adapt/StepReviewChanges.tsx`** -- Step 4: review AI-proposed changes with accept/reject
6. **`src/components/pricing/adapt/StepFinalConfirm.tsx`** -- Step 5: second-pass result and create button
7. **`src/components/pricing/adapt/VoiceInputButton.tsx`** -- Microphone button using native Web Speech API
8. **`supabase/functions/generate-adapt-questions/index.ts`** -- AI edge function that analyses base proposal and generates targeted questions
9. **`supabase/functions/adapt-pricing-proposal/index.ts`** -- Main AI edge function that takes base proposal + user inputs and produces the adapted item list with change rationale. Handles both initial generation and the refinement pass.

### Modified Files

1. **`src/pages/MatterPricing.tsx`** -- Add "New from Precedent" button next to "New Proposal"
2. **`src/App.tsx`** -- Add route `/pricing/adapt` for the wizard page
3. **`src/lib/hooks/usePricingProposals.ts`** -- No changes needed; the wizard will use the existing `createProposal` mutation and then bulk-insert items via a new `createProposalWithItems` mutation added here
4. **`supabase/config.toml`** -- Register the two new edge functions with `verify_jwt = false`

### Voice Input Implementation
Uses the browser-native Web Speech API (`SpeechRecognition`), which requires no API keys and works in Chrome, Edge, and Safari. A simple `VoiceInputButton` component will:
- Toggle recording on/off
- Show a pulsing microphone icon while recording
- Append interim and final transcripts to the text area
- Show a "not supported" message in Firefox (which lacks this API)

### AI Edge Function Design

**`generate-adapt-questions`**: Receives the base proposal's items, phases, and scope assumptions. Uses Lovable AI (`google/gemini-3-flash-preview`) with tool calling to return a structured array of questions, each with: question text, input type (text/select/boolean/checklist), and options where applicable.

**`adapt-pricing-proposal`**: Receives the full base proposal data, structured answers, free-form description, and (on second pass) user accept/reject decisions and comments. Returns a structured list of changes using tool calling:
```
{
  phaseChanges: [{ basePhaseId, action, newName?, rationale }],
  itemChanges: [{ baseItemId?, action, newWorkItem?, newDetail?, newCategory?, newPhaseId?, rationale }],
  scopeAssumptionChanges: [{ type, description, rationale }]
}
```

### Proposal Creation Flow
When the user confirms the final draft, the wizard:
1. Calls `createProposal` with client, name, currency, rate card (copied from base)
2. Inserts all adapted items into `pricing_proposal_items` for the new version
3. Saves adapted phases and scope assumptions to the proposal's JSONB columns
4. Navigates to the new proposal detail page
